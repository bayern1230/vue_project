布局

文档流(normal flow):网页是一个多层结构，通过CSS可以为每一层设置一个样式，用户只能看到最顶上的一层
这些层中，最底下的一层称为文档流
元素在文档流和不在文档流是两种不同的状态

块元素：

  • 在页面中独占一行
  • 默认宽度为父元素的全部宽度
  • 默认高度为子元素撑开
行内元素:
  • 行内元素不会独占一行
  • 行内元素自左向右水平排列
  • 行内元素的默认宽度和高度都是被内容撑开

盒子模型(box model)

  • 内容区(content)
  • 内边距(padding)
  • 边框(border)
  • 外边距(margin)

内容区

width 设置内容区的宽度，height 设置内容区的高度

外边距

  • border-width 边框宽度 border-xxx-width 可以分别为上下左右设置边框
  • border-color 边框颜色
  • border-style 边框样式
    • solid:实线
    • dotted:点状虚线
    • dashed:线状虚线
    • double:双线

盒子的可见框大小由content+padding+border大小决定

外边距会影响元素的位置，可以设置为负值使其向相反的方向移动

margin-left+border-left+padding-left+width+padding-right+border-right+margain-right = 父元素内容区宽度

如果上式不成立，则称为过度约束
如果没有auto的情况，则会自动添加margin-right,如果有auto，则会自动调整auto对应的值
经常利用这一点使得在父元素中内容居中
实例：width:xxpx;margin:0 auto

溢出

如果子元素在父元素中溢出，则使用overflow处理溢出的情况

  • visible:默认值，可见
  • hidden:溢出部分会被裁剪
  • scroll:滚动条
  • auto:根据需要生成滚动条

外边距的折叠

相邻的垂直方向外边距会发生重叠现象
兄弟元素间的相邻垂直外边距会取两者之间的较大值(绝对值)
如果一正一负，取两者之和

父子元素的相邻外边距，子元素会传递给父元素

行内元素的盒模型

行内元素不支持设置宽度和高度

行内元素可以设置padding和border,但是垂直方向不会影响布局

display & visibility

display:设置元素显示的类型

    可选值：
  • inline:将元素设置为行内元素
  • block:将元素设置为块元素
  • inline-block:行内块元素,既可以设置高度和宽度，又不会独占一行
  • table:将元素设置为一个表格
  • none:元素不在页面中显示

visibility:用来显示元素的状态

  • visible:默认值，正常显示
  • hidden:隐藏，但是依然占据位置

浮动

通过浮动可以使一个元素在其父元素之内左右移动

用float属性设置元素浮动

  • none:默认值，不浮动
  • left:向左浮动
  • right:向右浮动

注意：设置浮动之后水平布局的等式不需要强制成立

元素设置浮动之后，会完全从文档流脱离

浮动的特点:

  1. 浮动元素会完全脱离文档流
  2. 设置浮动以后元素会向父元素的左侧或者右侧移动
  3. 浮动元素默认不会从父元素中移出
  4. 浮动元素左右移动时，不会遮盖住其他的浮动元素
  5. 如果浮动元素上面是一个没有浮动的元素，则浮动元素无法上移
  6. 浮动元素不会超过它上边的浮动的兄弟元素
  7. 浮动元素不会盖住文字，文字会自动环绕在浮动元素周边
脱离文档流的特点:
  1. 块元素不再独占文档一行
  2. 宽度和高度默认被内容撑开
  3. 行内元素浮动之后特性与块元素相同

简单布局

高度塌陷

一般父元素高度不会写死，会随着内容的变化而变化

在浮动布局中，父元素高度默认是被子元素撑开的，当子元素浮动后，会完全脱离文档流 将会无法撑起父元素，导致父元素丢失
父元素高度丢失之后，会导致下面元素跟着上移，导致页面布局混乱

BFC(block formatting context)--块级格式化环境

BFC是CSS中的一个隐含属性，开启之后该元素会变成一个独立的布局区域

元素开启BFC的特点

  • 开启BFC的元素不会被浮动元素所覆盖
  • 开启BFC的元素子元素和父元素外边距不会重叠
  • 开启BFC的元素可以包含浮动的子元素

常用方式：为元素设置overflow:hidden

定位

positon:

  • static:默认值，元素静止，没有开启定位
  • relative:元素的相对定位
  • absolute:元素的绝对定位
  • fixed:元素的固定定位
  • sticky:元素的粘滞定位

相对定位(positon:relative)

元素开启相对定位之后，若不设置偏移量，元素不会发生任何变化

绝对定位

开启绝对定位之后，元素会从文档流中脱离

行内变成块，宽高被内容撑开

包含块:
默认情况下，包含块即离它最近的祖先元素
绝对定位的包含块：离它最近的开启了定位的祖先元素

元素层级

对于开启了定位的元素，可以通过z-index指定元素的层级,z-index需要一个整数参数，值越大，层级越高

祖先元素不会盖住后代元素
